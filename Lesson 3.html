What is HTML

  1. The heart of the web
  2. Made of text content (what you see)
  3. References to other documents (images and video)
  4. Links to other pages
What is HTML Markup?
  • Made up of:
    1. Tags: Contents
    2. First tage is the opening tag
    3. The whole thing is called an element
  • First tag is the b tag for making text bold. It starts with a b and ends with a b
  • The next tag is the em tag for making words italic; ends with the em tag and so forth
Computers are Stupid
  • This refers to how computers understand the instructions that human programmers have to give them
  • In spoken language, sentences often have a great deal of ambiguity. For example, in the sentence "Each of us saw her duck", does that mean we all saw her pet duck, or did we see her duck under something. Depending on the context, either could be correct
  • Computers can't make such inferences and only interpret language literally. Computer programming languages are designed to limit ambiguity, but sometimes this can result in issues between the programmer and the machine
  • They are thus very sensitive to typos and errors:
    1. What happens when we forget to add a closing em tag: Everything after the opening tag is italicised

HTML Attributes

img - images
  • An example is:
    • img src=url alt=text
      • Src (source) is the attribute, the URL to download from
      • The other attribute is alt, meaning alternet. It's what is displayed when the image can't load for whatever reason. It's required in the sense that HTML parses will complain if it's not there. It's used for broken requests
    • Img tags don't have closing tags, as there's no content. They're void tags
It's OK to Forget
  • Why is it that anchor tags use the href tag as an attribute whilst img tags use src? Remember, if you use one attribute in place of the other, your html won't work
  • It's okay to forget what to use where; that's what Google is for. Programming is about understanding the bigger concepts. Unless you write HTML regularly, you're bound to forget something soon. There are simply too many facts to remember them all
  • As suggested, Google is a good source for help. Also look at the website for W3Schools. Also check out MDN (Mozilla Developer Network)
  • The bottom line: Use documentation
Whitespace
  • Remember, that when text is input into HTML on multiple lines, it renders them on a single line
  • You can use the br tag to break one line into multiple lines. Br doesn't need a closing tag; it's another void tag
Paragraph Tag
  • Another way to break spae is by the p tag (paragraph). The

    tag does need a closing tag. It will render a line as one paragraph. With the p tag, there is an extra bit of space between the lines that doesn't occur when using the br tag

Inline vs Block Elements
  • Take the sentence: "This text is really too long one line"
  • If we format it as:
    • "This text     is br really 0   too       long   one   line"
  • It will always be rendered as:
    • This text is
    • Really too long one line
  • Regardless of the white space. Why is this? Why do we have two ways of making new lines? The br tag simply ends a line, so if we put it in between words, whatever is beyond the br tag simply wraps to the next line
  • The p tag works differently: It makes an invisible box around the text, which can have height and width attributes
  • Other inline elements include em and img
Span and Div Elements
  • Normal elements; both have content
  • Span is inline; Div is block
  • Take Span as an example:
    • <span> This is one line <span>
      Yep, one line
  • If we change it to:
    • <div> This is one line <div>
      Yep, one line
  • It behaves more like the <p> tag; the div creates a box around "This is one line"
  • Which elements are 'inline':
    • <a>
    • <span>
    • <br> (seems confusing, as it shigts words to a new line, but it doesn't behave like true block elements, in that it doesn't add height or width to anything, it simply shifts lines)
    • <img > (which is both inline and block)
    • <strong>

Document Structure

  • Here is the basic structure of an HTML document:
    <!DOCTYPE HTML> -doctype
    <html>
    <head>
    </head>
    <body> <b> content </b> </body> </html>
    • <!DOCTYPE HTML>: The attribute of 'html' used to be a lote more convoluted; this is not the case for HTML5, which just as the string 'html'
    • <html>: The opening html tag, surrounding the rest of the document
    • <head>: This tag contains meta-data and other things. If you're using CSS or Javascript, it goes in this tag
    • <title>: The title of the tag, as it would appear at the top of a browser window
    • <body>: Contains the contents of the documents. We've been working with contents of the body tag so far

Summary

  • The web is largely comprised of HTML documents (hypertext markup language). HTML is the language browsers use to interpret web pages. The components of the web include the client (your PC and browser), the Internet and servers. The client and server interact via http (hypertext transfer protocol)
  • HTML elements are what we use to tell a browser how to display content on pages. Most HTML elements consist of an opening and closing tag with come content in between
  • Some elements accept additional values called attributes in their opening tags. These provide additional info to the browser. For example, the anchor takes an attribute called href and defines a URL to link to
  • Other HTML elements include:
    • <p>: Defines a paragraph. The content of the <p> tag is in a 'box'
    • <bt>: Creates a line
    • <div>: Creats a 'box' container element
    • <span>: Creates an 'inline' container element
